Supapoker deployment checklist

Cloudflare setup guide

Use this guide to prepare the Cloudflare resources needed by this NuxtHub version of Supapoker, wire the production configuration, and deploy the Worker.

Nuxt 4 NuxtHub D1 R2 Blob Durable Objects Better Auth Cloudflare Email

Repo Status

What is already present versus what still needs your Cloudflare account values.

Already configured: Worker build, Durable Object binding, email binding, Better Auth runtime variables, NuxtHub modules.
You still provide: D1 database ID, R2 bucket name, production domain, GitHub OAuth app, email sender, secrets.
1

Install and Login

Install dependencies, then authenticate Wrangler against the Cloudflare account that will own the app.

pnpm install
pnpm exec wrangler login
pnpm exec wrangler whoami
Use the same Cloudflare account for D1, R2, Email Sending, and the deployed Worker unless you intentionally separate accounts.
2

Create Cloudflare Resources

Create the production D1 database and R2 bucket. Save the D1 database_id printed by Wrangler.

pnpm exec wrangler d1 create supapoker-prod
pnpm exec wrangler r2 bucket create supapoker-avatars
  • D1 stores rooms, profiles, stories, historical estimates, and Better Auth tables.
  • R2 stores uploaded profile images through NuxtHub blob.
  • Durable Objects are created by deployment migration, so there is no separate create command.
3

Wire NuxtHub to D1 and R2

The app currently enables NuxtHub database and blob support. Before production deploy, replace the placeholder-style local configuration in nuxt.config.ts with your D1 database ID and R2 bucket name.

hub: {
  db: {
    dialect: "sqlite",
    driver: "d1",
    connection: {
      databaseId: "<D1_DATABASE_ID_FROM_WRANGLER>"
    }
  },
  blob: {
    driver: "cloudflare-r2",
    bucketName: "supapoker-avatars"
  },
  kv: false,
  cache: false
}
Do not edit files inside .output. Build output is generated by Nuxt/Nitro and should be recreated with pnpm run build.
4

Confirm Worker Bindings

The root wrangler.jsonc already includes the Durable Object binding and the Cloudflare Email binding.

{
  "name": "supapoker",
  "durable_objects": {
    "bindings": [
      { "name": "ROOM_SESSION", "class_name": "RoomSession" }
    ]
  },
  "migrations": [
    { "tag": "v1", "new_sqlite_classes": ["RoomSession"] }
  ],
  "send_email": [
    { "name": "EMAIL" }
  ]
}

Optional: restrict allowed sender addresses after your email sender is verified.

"send_email": [
  {
    "name": "EMAIL",
    "allowed_sender_addresses": ["noreply@your-domain.com"]
  }
]
5

Set Email Sending

In the Cloudflare dashboard, configure Email Sending for the domain you will send from.

  • Verify the domain or sender address in Cloudflare Email.
  • Use a real sender such as noreply@your-domain.com.
  • Set EMAIL_FROM to a value Better Auth can use in transactional emails.
SupaPoker <noreply@your-domain.com>
If emails are skipped in production logs, confirm the deployed Worker has a binding named exactly EMAIL.
6

Create the GitHub OAuth App

Create an OAuth app in GitHub Developer Settings using your final production URL.

Field Value
Application name Supapoker
Homepage URL https://your-domain.com
Authorization callback URL https://your-domain.com/api/auth/callback/github
The callback URL must match exactly. A wrong scheme, host, path, or trailing environment domain will break GitHub login.
7

Set Production Secrets and Variables

Generate a Better Auth secret, then add secrets to the deployed Worker. Do not commit real values to the repository.

openssl rand -base64 32

pnpm exec wrangler secret put BETTER_AUTH_SECRET
pnpm exec wrangler secret put GITHUB_CLIENT_ID
pnpm exec wrangler secret put GITHUB_CLIENT_SECRET

Add non-secret runtime variables through the Cloudflare dashboard or a vars block in wrangler.jsonc.

"vars": {
  "NUXT_PUBLIC_SITE_URL": "https://your-domain.com",
  "EMAIL_FROM": "SupaPoker <noreply@your-domain.com>"
}
Name Purpose Recommended storage
BETTER_AUTH_SECRET Signs Better Auth sessions and tokens. Secret
GITHUB_CLIENT_ID GitHub OAuth app identifier. Secret or variable
GITHUB_CLIENT_SECRET GitHub OAuth app secret. Secret
NUXT_PUBLIC_SITE_URL Public app origin used for auth redirects and links. Variable
EMAIL_FROM Verified sender used for transactional email. Variable
8

Build and Deploy

Run checks locally, build the Nuxt app, then deploy the generated Cloudflare Worker.

pnpm run typecheck
pnpm run build
pnpm run cf:deploy

The cf:deploy script runs pnpm build and then wrangler --cwd .output deploy.

After build, the Cloudflare deployment config lives at .output/server/wrangler.json. Inspect it if a binding is missing.
9

Verify D1 Tables

NuxtHub includes database migrations in the build. After deploy, confirm the D1 database has the expected tables.

pnpm exec wrangler d1 execute supapoker-prod \
  --remote \
  --command "SELECT name FROM sqlite_master WHERE type='table';"
If the tables are missing, run the NuxtHub migration command for the configured production database before testing auth or room creation.
10

Add the Custom Domain

In Cloudflare Workers, attach your custom domain or route to the supapoker Worker.

  • Worker target: supapoker
  • Domain example: poker.your-domain.com
  • Update NUXT_PUBLIC_SITE_URL to the final URL.
  • Update the GitHub OAuth callback to the final URL.
11

Production Smoke Test

GitHub login completes and returns to the app.
A signed-in user can create a room.
A second signed-in user can join from an invite link.

12

Troubleshooting Map

Symptom Likely cause Fix
Room realtime binding is not configured The deployed Worker output is missing ROOM_SESSION. Run pnpm run build and inspect .output/server/wrangler.json. Confirm root wrangler.jsonc has the Durable Object binding and migration.
D1 binding or database errors NuxtHub production database config is missing the D1 database ID. Add the D1-backed hub.db config in nuxt.config.ts, rebuild, and redeploy.
Email is skipped or unavailable The deployed Worker does not have an EMAIL send_email binding, or the sender is not verified. Verify Cloudflare Email setup and inspect the generated Wrangler output for send_email.
GitHub login redirect mismatch The GitHub OAuth callback URL does not exactly match production. Set callback to https://your-domain.com/api/auth/callback/github.
Auth links point to the wrong host NUXT_PUBLIC_SITE_URL is missing or still set to localhost. Set it to the final production URL and redeploy.
